Getting Started with Debug Server Scripting

Overview

Debug Server Scripting (DSS) is a set of cross platform Java APIs to the Debug Server which allow scripting through Java or 3rd Party tools such as JavaScript (via Rhino), Perl (via Inline::Java), Python (via Jython), TCL (via Jacl/Tclblend), etc. JavaScript is the default (and preferred) scripting language supported by DSS.

The Debug Server provides debugger functionality to both DSS and the Eclipse GUI. DSS is lightweight as it can be used indepdendent of the GUI.

Resources

  • Debug Server Scripting Mediawiki: This site is your first stop for DSS information and material. It contains valuable information on how to use DSS. 
  • DSS API Documentation: The complete DSS API list is split between two documents – Debug Server (DS) API documentation and DVT (Data Visualization Toolkit) API documentation:
    • DS API Documentation: detailed reference documentation for the main debugger APIs available from DS. This includes APIs for creating the main scripting environment, all basic debugging actions, enabling/disabling logging, and setup of profiling actions.
    • DVT API Documentation: detailed reference documentation for the data visualization and analysis APIs available from DVT. APIs for formatting and exporting collected profiling data are available here.
  • The DSS Examples make a good starting point to see how the DSS API is used within a script.
    • DSS Tutorial: A tutorial for beginners
    • DS Examples: various scripts which perform basic debug operations on a C64x+ simulator. Run go.bat or go.sh to execute the examples.
    • DVT Examples: various scripts demonstrate how to use the DVT profiling APIs from a script. Execute these using go.bat or go.sh.
  • Release Notes: These notes document some of the known issues with this release as well as any workarounds that are present.
