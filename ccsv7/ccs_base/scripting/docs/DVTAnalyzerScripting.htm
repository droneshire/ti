Getting Started with DVT Analyzer Scripting

Introduction sfsdf

DVT Analyzer Scripting is a subset of the DVT scripting available via the Debug Server Scripting (DSS) environment. See Getting Started with DSS and DVT scripting API Document for full set of APIs (including API for Simulation Profiler).

Analyzer Scripting is currently supported for Hardware Trace Analyzer, RTOS Analyzer and System Analyzer.

  • DVT scripting API Document
  • Hardware Trace Analyzer User Guide and System Analyzer User Guide available in Code Composer Studio Help
  • Hardware Trace Analyzer Wiki Page

Terminology

Analysis Configuration

(aka Usecase)

It is the list of features available in Code Composer Tools menu under Hardware Trace Analyzer, RTOS Analyzer and System Analyzer (e.g. Code Coverage, Memory Throughput Analysis and Statistical Profiler)

An analysis configuration takes care of setting-up the target to collect the required data, launching the required Data Provider to collect, decode and display the logs, and hooking up the appropriate Analyzer to perform the required analysis

Users can modify an existing Analysis Configuration and save it as a User Configuration. User Configuration can be scripted similarly to Factory Configuration.

Data Provider

Data Provider collect, decodes and display events in the events log table (i.e. Trace Viewer for Trace or Live Session view for System/RTOS Analyzer).

Hardware Trace Analyzer uses the Trace Data Provider. System and RTOS Analyzer uses the UIA Data Provider.

Data Providers are automatically started by the Analysis Configuration.

Analyzer

(aka Analysis Feature)

Analyzers are used for analyzing data available from the Data Provider. Example the Code Coverage Analyzer performs Function and Line coverage analysis, Function Profiler determines how many cycles are spent in each function, etc.

Analysis Configurations may automatically start some Analyzers. Analyzer can also be explicitly launched from the Data Provider log view toolbar (i.e. Trace Viewer or Live Session).

Overview

It is recommended to become familiar with the available Analysis Configurations and their properties via Code Composer Studio GUI before scripting.

Tip: It may be easier to create and save configuration in CCStudio GUI for scripting than configuring the analysis via scripting. User saved configurations are scriptable the same way factory configurations are.

Here is an example of the steps involved in a typical scripting session (not including the debug server steps for loading, running the application, etc.):

·         Open a DVT Analysis Session

·         Load the required Analysis Configuration

o   Table below list the names of the available Factory Analysis Configurations available

o   Names of Analysis Configurations can also be obtained for CCStudio Tools menu

o   User saved configurations can also be scripted

·         Set required properties of the Analysis Configuration (if not using defaults)

o   E.g. for hardware trace analyzer, select the CPU and receiver (if not using defaults)

o   See table below for list of properties and values that can be modified via scripting

o   Properties can also be saved to User Configuration via CCStudio GUI

·         Run the Analysis Configuration

·         Get Handle to Analyzer and/or Data Provider

o   Handle is obtained using the name of the Analyzers and/or Data Provider

o   See table below for list of Analyzer and Data Provider names

o   Analyzer names can also be obtained from the toolbar of the Data Provider log view (e.g Trace Viewer or Live Session)

o   Scripting support running additional Analyzers that are not part of the Analysis Configuration

·         Wait for the analyzer to finish processing data

·         Export data from required views

o   Data is exported by specifying the name of the buffer the view is connected to

o   See table below for buffer names

o   Note that log views (Trace Viewer and Live Session) do not belong to analyzers, they belong to Data Providers

See this folder for scripting examples: /ccs_base/scripting/examples/DVTExamples

Hardware Trace Analyzer scripting example:

analysisSession = dvtServer.openAnalysisSession();

...

analysis = analysisSession.loadAnalysis("Hardware Trace Analyzer/Code Coverage");

...
analysis.setProperty("cpu", "Spectrum Digital XDSPRO USB Emulator_0/C66xx_0");

analysis.setProperty("receiver", "Pro_Trace");

analysis.setProperty("buffersize", "64 kB");

analysis.setProperty("merge", true);
...
analysis.run();
...
analyzer = analysis.findAnalyzerByName("Code Coverage);

dataProvider = analysis.findDataProviderByName("Trace Viewer);

 

analyzer.waitForEndOfData();

 

analyzer.exportDataToCSV("Line Coverage", "/LineCoverage.csv", null);

dataProvider.exportDataToCSV("Trace Viewer", "/TraceLog.csv", null);

analysisSession.endAnalysis(analysis);
